# Feature Specification: 手機 UX 優化第一批（自動同步、即時導覽、位置瀏覽、返回鍵）

**Feature Branch**: `002-mobile-ux-uplift`

**Created**: 2026-07-14

**Status**: Draft

**Language**: zh-TW

**Input**: User description: "手機 UX 優化第一批：補完既有體驗承諾。範圍四項：(1) 自動同步；(2) 移除路由人為延遲；(3) 位置詳情瀏覽動線；(4) 非頂層路由返回鍵。所有 UI 文案使用繁體中文，維持離線優先：所有功能離線可用，雲端不可用時不阻塞本地流程。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 家庭資料自動同步 (Priority: P1)

家庭成員 A 在自己的手機新增或移動物品後，成員 B 打開（或切回）App 時，不需要任何人進設定頁按「立即同步」，就能看到最新的家庭庫存。使用者平常完全不需要意識到「同步」這件事；只有在想確認狀態時，首頁會告訴他目前是否已同步完成。

**Why this priority**: 目前同步只能在設定頁手動觸發，家庭共享形同失效——這是本產品「家庭共享庫存」定位的最大缺口，價值最高。

**Independent Test**: 兩個瀏覽器以同帳號登入同家庭；A 新增一筆物品後，B 只做「切回 App / 重新開啟 App」動作，能在合理時間內看到該物品，全程不進設定頁。

**Acceptance Scenarios**:

1. **Given** 使用者已登入且有網路，**When** 本地完成任一會產生待同步作業的操作（新增/編輯/移動/刪除物品、位置、照片等），**Then** 系統在短暫延遲內自動推送待同步作業，不需手動操作。
2. **Given** App 剛啟動且已登入並選定家庭，**When** 進入首頁，**Then** 系統自動拉取一次遠端變更。
3. **Given** App 在背景，**When** 使用者切回前景，**Then** 系統自動拉取遠端變更（受最小間隔限制，避免頻繁切換造成重複請求）。
4. **Given** 裝置離線且累積了待同步作業，**When** 網路恢復，**Then** 系統自動推送並拉取，不需使用者操作。
5. **Given** 首頁顯示同步狀態指示，**When** 使用者點擊該指示，**Then** 立即觸發一次同步，且同步中/成功/失敗有明確視覺回饋。
6. **Given** 自動同步失敗（伺服器錯誤或斷線），**When** 使用者繼續操作 App，**Then** 本地操作完全不受阻塞，不彈出干擾性錯誤視窗；失敗以低干擾方式呈現於同步狀態指示，並依既有退避機制稍後重試。
7. **Given** 未設定雲端（純離線模式），**When** 使用者操作 App，**Then** 自動同步不啟動、不顯示任何同步錯誤。

### User Story 2 - 打開位置看裡面有什麼 (Priority: P2)

使用者想知道「儲藏室的第三個箱子裡有什麼」。他打開位置頁，點該箱子的名稱，直接看到箱子（含其子位置）裡所有物品的照片卡片清單；也可以從這裡直接「在此位置新增物品」，或進入編輯該位置。

**Why this priority**: 「打開 App → 找到容器 → 看裡面有什麼」是收納 App 最核心的查詢動線，目前只能繞道搜尋頁的進階篩選達成。

**Independent Test**: 建立一個含子位置與物品的位置樹；從位置頁點位置名稱，兩次點擊內看到該位置與子位置的物品清單，並能由此進入新增物品（位置已預選）。

**Acceptance Scenarios**:

1. **Given** 位置頁顯示位置樹，**When** 使用者點某位置的名稱，**Then** 進入該位置的詳情頁，顯示位置名稱、類型、完整路徑，以及該位置（含所有子位置）內的物品卡片清單。
2. **Given** 位置詳情頁，**When** 物品屬於子位置，**Then** 卡片上可辨識其實際所在的子位置。
3. **Given** 位置詳情頁沒有任何物品，**When** 頁面載入完成，**Then** 顯示友善的空狀態文案與「在此位置新增物品」入口。
4. **Given** 位置詳情頁，**When** 使用者點「在此位置新增物品」，**Then** 進入新增物品頁且位置欄位已預選為該位置。
5. **Given** 位置詳情頁，**When** 使用者點「編輯位置」，**Then** 可修改該位置（沿用既有編輯能力）。
6. **Given** 位置頁的位置節點，**When** 使用者想編輯位置，**Then** 每個節點有明確的編輯按鈕（點名稱＝進詳情，點編輯鈕＝編輯）。
7. **Given** 使用者以深連結直接開啟一個已被刪除或不存在的位置詳情，**Then** 顯示「找不到位置」的說明並提供回位置頁的入口。

### User Story 3 - 頁面切換即時反應 (Priority: P3)

使用者在底部導覽切換頁面時，頁面立即出現，不再每次都先看到一段骨架載入動畫。

**Why this priority**: 目前每次導覽被人為加上約 220ms 的假載入延遲；本地 SPA 實際是即時的。移除後每一次操作都變快，改動成本極低。

**Independent Test**: 點擊底部導覽任一分頁，目標頁面內容立即渲染，無過渡骨架畫面。

**Acceptance Scenarios**:

1. **Given** 使用者在任一頁，**When** 點擊底部導覽或任何站內連結，**Then** 目標頁面立即顯示，無人為延遲與骨架過場。
2. **Given** 頁面切換後，**When** 新頁面顯示，**Then** 捲動位置回到頂部（維持既有行為）。

---

### User Story 4 - 非頂層頁面有返回鍵 (Priority: P4)

使用者把 App 加入 iOS 主畫面以 standalone 模式使用（沒有瀏覽器返回 UI）。他從首頁點進一個物品的詳情頁後，畫面左上角有返回鍵可以回到上一頁，不會被困在頁面裡。

**Why this priority**: standalone 模式下非頂層頁（物品詳情、回收桶、位置詳情）目前沒有畫面內返回手段，屬於可能的死路；影響面明確但頁面數量少。

**Independent Test**: 在視窗尺寸模擬手機的瀏覽器中進入物品詳情頁，header 顯示返回鍵，點擊後回到來源頁；直接以深連結開啟詳情頁（無瀏覽歷史）時，返回鍵仍能回到合理的上層頁。

**Acceptance Scenarios**:

1. **Given** 使用者由站內導覽進入物品詳情／回收桶／位置詳情頁，**When** 點擊 header 返回鍵，**Then** 回到上一頁。
2. **Given** 使用者以深連結（或冷啟動）直接開啟非頂層頁、沒有站內瀏覽歷史，**When** 點擊返回鍵，**Then** 導向該頁的預設上層頁：物品詳情→首頁、位置詳情→位置頁、回收桶→設定頁。
3. **Given** 使用者在頂層頁（首頁、位置、新增、搜尋、設定），**Then** header 不顯示返回鍵。

### Edge Cases

- 自動同步進行中，使用者又點首頁指示或設定頁「立即同步」：同一時間僅允許一個同步流程，重複觸發被忽略或合併，不會並發推送。
- 同步進行到一半網路中斷：該次同步靜默失敗，狀態指示顯示待同步數，下次觸發（回前景／上線／寫入）再自動重試。
- 使用者登出或切換家庭：既有的自動同步排程停止，不得把前一個家庭的作業推到新的上下文。
- 位置詳情頁的位置在其他裝置被刪除後同步下來：重新整理或返回時顯示「找不到位置」而非錯誤畫面。
- 位置內物品數量很大：清單仍可正常捲動瀏覽（分頁／虛擬化不在本批範圍）。
- 快速連續點擊底部導覽多個分頁：以最後一次點擊為準，無殘留過場狀態。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 本地完成任一會產生待同步作業的操作後，系統 MUST 在短暫去抖動延遲內自動嘗試推送待同步作業，無需使用者手動操作。
- **FR-002**: App 啟動完成且已登入並選定家庭時，系統 MUST 自動執行一次遠端變更拉取。
- **FR-003**: App 由背景回到前景時，系統 MUST 自動拉取遠端變更，並以最小間隔節流避免重複請求。
- **FR-004**: 網路由離線恢復為連線時，系統 MUST 自動推送待同步作業並拉取遠端變更。
- **FR-005**: 自動同步的任何失敗 MUST 不阻塞本地操作、不彈出干擾性錯誤視窗；沿用既有的每筆作業退避重試機制。
- **FR-006**: 首頁 MUST 顯示同步狀態指示：包含待同步筆數與最近一次同步結果（進行中／成功／失敗），點擊可立即觸發同步。
- **FR-007**: 設定頁 MUST 保留手動「立即同步」，並與自動同步共用防重入保護（同一時間僅一個同步流程）。
- **FR-008**: 未設定雲端（純離線模式）時，自動同步 MUST 完全不啟動，且不顯示同步錯誤或提示。
- **FR-009**: 使用者登出或切換家庭時，系統 MUST 停止原有的自動同步排程並重新綁定至新的登入／家庭狀態。
- **FR-010**: 站內頁面切換 MUST 即時完成，移除固定的人為延遲與過場骨架；切換後維持捲動回頂部的既有行為。
- **FR-011**: 系統 MUST 提供位置詳情頁（路由 `/locations/:id`）：顯示位置名稱、類型、完整路徑，以及該位置與其所有子位置內的物品卡片清單；子位置物品需可辨識實際所在位置；無物品時顯示空狀態。
- **FR-012**: 位置詳情頁 MUST 提供「在此位置新增物品」入口，進入新增物品頁時該位置已預選；並提供編輯該位置的入口。
- **FR-013**: 位置頁的每個位置節點 MUST 改為：點名稱進入詳情頁、另設明確的編輯按鈕；刪除按鈕維持既有行為。
- **FR-014**: 非頂層路由（物品詳情、位置詳情、回收桶）header MUST 顯示返回鍵；有站內瀏覽歷史時返回上一頁，無歷史時導向預設上層頁（物品詳情→首頁、位置詳情→位置頁、回收桶→設定頁）；頂層路由不顯示返回鍵。
- **FR-015**: 本功能所有新增或修改的 UI 文案 MUST 使用繁體中文；所有功能 MUST 在離線狀態可用（同步相關指示在離線時如實呈現待同步狀態）。

### Key Entities

- 本功能不新增資料實體。同步狀態為執行期 UI 狀態（待同步筆數、最近結果），來源為既有的待同步作業佇列；位置詳情頁使用既有的位置與物品資料。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 兩台裝置登入同一家庭且皆有網路時，一方完成的變更，另一方在「切回前景或重新開啟 App」後 10 秒內可見，全程無人操作同步。
- **SC-002**: 底部導覽切換頁面無可感知的人為延遲（無過場骨架；本地資料下切換體感即時）。
- **SC-003**: 使用者從位置頁出發，最多 2 次點擊即可看到任一位置（含子位置）內的物品清單。
- **SC-004**: 以 standalone 模式（無瀏覽器返回 UI）操作時，所有非頂層頁面皆可透過畫面內返回鍵回到上層頁，不存在無法離開的頁面。
- **SC-005**: 離線狀態下新增／編輯／移動／瀏覽全部照常可用，且不出現任何阻塞性同步錯誤視窗。

## Assumptions

- 自動推送的去抖動延遲採 2–3 秒等級的合理預設；回前景拉取的節流間隔採 30 秒等級的合理預設，實作時可微調。
- 首頁同步狀態指示只呈現摘要（筆數與最近結果）；詳細失敗原因清單仍保留在設定頁的同步區塊。
- 衝突偵測與衝突佇列沿用既有機制，本批不改變衝突處理策略。
- 照片本體（Blob）跨裝置同步不在本批範圍（既知缺口，屬後續批次）。
- 位置詳情頁預設同時列出直屬物品與所有子位置物品；清單量大時的分頁／虛擬化不在本批範圍。
- 返回鍵的「有無站內歷史」以本次 App 工作階段內的站內導覽為準；跨工作階段不追蹤。
